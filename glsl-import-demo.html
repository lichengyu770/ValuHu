GLSL Shader Import Demo

Note: This demo shows how to properly import GLSL shader files using the ?raw suffix in a Vite project.

Import Syntax for GLSL Shaders

JavaScript Import Syntax

// Import GLSL file with ?raw suffix to get the raw string content
import shaderString from './shader.glsl?raw';

// Now you can use the shader string with WebGL/Three.js
const fragmentShader = gl.createShader(gl.FRAGMENT_SHADER);
gl.shaderSource(fragmentShader, shaderString);
gl.compileShader(fragmentShader);

Implementation Steps

  1. Update Vite Configuration
    • Add assetsInclude: ['**/*.glsl', '**/*.vert', '**/*.frag'] to handle GLSL files
  2. Create GLSL Shader Files
    • Create separate files for vertex shaders (.vert) and fragment shaders (.frag)
    • Or use .glsl extension for general shader code
  3. Import the Shader with ?raw Suffix
    • This tells Vite to import the file as a raw string instead of processing it
  4. Use the Shader String with WebGL
    • Pass the imported string to WebGL shader compilation functions

Example GLSL Shader

shader.glsl

// Simple fragment shader example
precision mediump float;

// Input from vertex shader
varying vec2 v_texCoord;

// Uniforms
uniform sampler2D u_texture;
uniform float u_time;
uniform vec2 u_resolution;

void main() {
  // Calculate normalized pixel coordinates
  vec2 uv = gl_FragCoord.xy / u_resolution.xy;
  
  // Create a simple wave effect
  float wave = sin(uv.x * 10.0 + u_time) * 0.05 + 
               cos(uv.y * 10.0 + u_time) * 0.05;
  
  // Apply the wave distortion
  vec2 distortedUV = uv + wave;
  
  // Sample the texture
  vec4 texColor = texture2D(u_texture, distortedUV);
  
  // Add a color tint that changes over time
  vec3 tint = vec3(
    0.5 + 0.5 * sin(u_time),
    0.5 + 0.5 * cos(u_time * 0.5),
    0.5 + 0.5 * sin(u_time * 0.8)
  );
  
  // Output the final color
  gl_FragColor = vec4(texColor.rgb * tint, texColor.a);
}

React Component Implementation

Component Using Imported Shader

import React, { useEffect, useRef } from 'react';

// Import shader with ?raw suffix
import shaderString from './shader.glsl?raw';

const ShaderExample = () => {
  const canvasRef = useRef(null);
  
  useEffect(() => {
    const canvas = canvasRef.current;
    if (!canvas) return;
    
    const gl = canvas.getContext('webgl');
    if (!gl) return;
    
    // Set up WebGL context
    canvas.width = 400;
    canvas.height = 300;
    gl.viewport(0, 0, canvas.width, canvas.height);
    
    // Create and compile shaders using the imported string
    const fragmentShader = gl.createShader(gl.FRAGMENT_SHADER);
    gl.shaderSource(fragmentShader, shaderString);
    gl.compileShader(fragmentShader);
    
    // Rest of WebGL setup code...
    // (vertex shader, program linking, attributes, uniforms, etc.)
    
  }, []);
  
  return ;
};

export default ShaderExample;

Vite Configuration

vite.config.js

// Add these to your existing Vite configuration
export default defineConfig({
  // ... existing config
  
  // Configure static asset handling for GLSL files
  assetsInclude: ['**/*.glsl', '**/*.vert', '**/*.frag', '**/*.worklet.js']
})
Why use ?raw suffix? GLSL shaders need to be processed as raw text strings, not as modules or assets. The ?raw suffix tells Vite to import the file content directly as a string, which is exactly what WebGL needs for shader compilation.
